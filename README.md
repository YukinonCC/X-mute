# 🔇 X (Twitter) 列表批量隐藏工具 (Phantom 终极穿透版)

一个运行在 Tampermonkey (油猴) 上的高级用户脚本。它能够在你浏览 X（原 Twitter）的“正在关注”或“关注者”列表时，提供**一键静音 (Mute)** 和**全页批量静音**的功能。

本脚本采用了特殊的**底层请求劫持技术**，完美突破了 X 最新的 `Code 32` 反机器人风控防线，是目前最稳定、最安全的批量隐藏解决方案。

## ✨ 核心特性

* 🛡️ **突破 Code 32 顶级风控**：独创 Phantom 拦截引擎，利用 `unsafeWindow` 穿透网页 CSP 限制，动态窃取浏览器合法生成的 `x-client-transaction-id` 护照，实现 100% 伪装成官方请求。
* 🟢 **可视状态雷达**：屏幕左下角自带悬浮雷达。精准提示脚本是否已经成功抓取到官方安全凭证，告别“盲猜”式点击。
* ⚡ **绝对不死 UI 注入**：摒弃脆弱的 DOM 监听器，采用高频轮询注入技术。无视 X 前端 React 框架的频繁重绘，确保“隐藏”按钮永远牢牢钉在用户名旁边。
* 📦 **双端持久化记忆**：
  * **云端**：真正调用 X 官方 API，用户会被永久加入你账号的“已静音列表”。
  * **本地**：利用浏览器 `localStorage` 记录已处理名单。刷新页面后，已静音用户自动变灰、半透明，不再显示操作按钮，防止重复点击。
* ⏱️ **安全节流机制**：批量执行时强制加入 `1.5秒/个` 的处理间隔，最大程度模拟真人操作节奏，保护账号安全。

## 🛠️ 安装指南

1. **安装浏览器扩展**：在你的浏览器（Chrome / Edge / Firefox）中安装 [Tampermonkey (油猴)](https://www.tampermonkey.net/) 插件。
2. **添加脚本**：
   * 点击浏览器右上角的 Tampermonkey 图标，选择 **“添加新脚本”**。
   * 清空编辑器里原有的全部代码。
   * 将本脚本的完整代码复制并粘贴进去。
   * 按下 `Ctrl + S` (或 `Cmd + S`) 保存。

## 🚀 使用教程

本工具的使用需要配合“欺骗”网页的特定操作，请务必仔细阅读以下步骤：

1. **进入列表页面**：打开 X 并进入任何一个用户的“正在关注 (Following)”或“关注者 (Followers)”列表页。
2. **观察状态雷达（关键！）**：
   * 页面刚加载时，左下角会显示红色的提示：`🔴 正在等待凭证... 请【往下滚动页面】加载新用户！`。
   * **此时不要点击任何隐藏按钮！**
3. **激活脚本**：
   * 拿起鼠标**往下滚动页面**，让列表刷出底部的新用户。
   * 就在加载新用户的瞬间，脚本会瞬间窃取官方请求头。左下角的雷达会立刻变绿：`🟢 官方安全凭证已截获！现在可点击隐藏。`
4. **开始操作**：
   * **单人静音**：点击用户名旁边的 `🔇 隐藏` 按钮。
   * **批量静音**：点击右下角悬浮的 `⚠️ 批量 Mute 当前页` 按钮。脚本将以安全的速度自动处理当前屏幕上所有未隐藏的用户。

## 🧠 原理与数据存储说明

**Q：被我隐藏的用户，数据存在哪里？重启电脑会失效吗？**

本工具采用**双重保险**的存储机制，重启电脑或更换网络均不会导致已隐藏的人复原：
1. **真实 API 静音（账号级）**：当你看到提示“✔️ 已隐藏”时，脚本已经向 X 服务器发送了真实的 Mute 指令。这个人已经被加入你账号的官方黑名单，其推文将永远从你的信息流中消失。
2. **本地视觉缓存（浏览器级）**：为了提升视觉体验并防止重复请求，脚本会将该用户名存入你当前浏览器的本地缓存。下次打开页面时，缓存里的人会直接呈现“变灰半透明”的视觉效果。

## ⚠️ 风控与安全警示

* **切勿过度滥用**：虽然本脚本加入了 1.5 秒的延时，并且完美伪装了请求头，但如果你在**短时间内（如半小时内）疯狂静音上千个账号**，依然可能触发 X 的高频操作风控，导致账号被要求进行人机验证（验证手机号或验证码）。
* **建议使用频率**：建议每次批量处理 100-200 人左右，随后正常浏览一会儿推文，分批次清理最为稳妥。

## 🐞 常见问题排错

* **Q：为什么雷达一直红着，怎么往下滚都不变绿？**
  * A：请确保你处于完整的列表页面中。如果网页滚动到底部没有新用户加载（无法触发网页的网络请求），脚本就无法抓取凭证。尝试换一个关注人数较多的账号列表进行测试。
* **Q：为什么点击了没反应，或者显示“❌ 失败”？**
  * A：如果发生失败，屏幕下方会自动弹出一个绿色的 `⚠️ 错误日志` 框。请查看里面的 `状态码`。如果是 `401` 或 `Code 32`，通常是因为页面停留时间过长导致凭证过期，**刷新页面**重新走一遍流程即可解决。
